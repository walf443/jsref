defineProperty

Introduced in JavaScript 1.9.3

This method is compliant with ECMAScript 5.

Summary

Defines a new property directly on an object, or modifies an existing property on an object, and returns the object.

Syntax

Object.defineProperty(obj, prop, descriptor)

Parameters

obj
The object on which to define the property.
prop
The name of the property to be defined or modified.
descriptor
The descriptor for the property being defined or modified.

Description

This method allows precise addition to or modification of a property on an object. Normal property addition through assignment creates properties which show up during property enumeration, whose values may be changed, and which may be deleted. This method allows these extra details to be changed from their defaults.

A property descriptor is an object with the following fields:

value
The value associated with the property. (data descriptors only)
writable
True iff the value associated with the property may be changed. (data descriptors only)
get
A function which serves as a getter for the property, or undefined if there is no getter. (accessor descriptors only)
set
A function which serves as a setter for the property, or undefined if there is no setter. (accessor descriptors only)
configurable
True iff the type of this property descriptor may be changed and if the property may be deleted from the corresponding object.
enumerable
True iff this property shows up during enumeration of the properties on the corresponding object.

Property descriptors present in objects come in two main flavors: data descriptors and accessor descriptors. A data descriptor is a property which has a value and which may or may not be writable. An accessor descriptor is a property described by a getter-setter pair of functions. A descriptor is one of these two flavors; it cannot be both. All descriptors regardless of flavor include the configurable and enumerable fields.

When the property specified doesn't exist in the object, Object.defineProperty creates a new property as described. Fields may be omitted from the descriptor, and default values for those fields are imputed. All of the boolean-valued fields default to false. The value, get, and set fields default to undefined. The property you specify can't have two fields which indicate different descriptor types, and a TypeError will be thrown if you specify such a property. For example, this code will throw an exception:

// throws: value appears only in data descriptors, get appears only in accessor descriptors
Object.defineProperty({}, "conflict", { value: 0x9f91102, get: function() { return 0xdeadbeef; } });

When the property already exists, Object.defineProperty attempts to modify the property according to the values in the descriptor. The precise details of exactly what modifications are permitted when are complex and not easy to describe, but the following rules mostly cover it:

  • If the old descriptor has the same fields and values as the new one (or the new one has a subset of the fields in the old one; note that this differs from how omitted fields are treated when the property didn't already exist), nothing happens. Object.defineProperty is idempotent.
  • If the old descriptor had its configurable field as false, then no fields present in the new descriptor can be different from those in the old descriptor. Further, the type of the new descriptor must be the same as the type of the old one.

A TypeError is thrown when any of the above failure cases occur.

One final case is worth examining more closely: that where the provided descriptor doesn't contain any fields indicating it to be a data descriptor or an accessor descriptor, which is to say its only fields as named above (unrelated properties may be present and would be ignored) are enumerable and configurable (one, both, or neither):

Object.defineProperty({}, "p", { enumerable: false });

If there is no p property already, the descriptor is treated as if it were a data descriptor, and omitted field values are imputed as described before. If there is such a property, an attempt is made to copy the field values in the new descriptor to the old one. If no fields are present, nothing happens. If fields are present, the old descriptor must have been configurable or the values to be copied over must be identical to those of the old descriptor.

Examples

var o, d;

o = {};
Object.defineProperty(o, "p", { value: 42 });
assert(o.p === 42);
for (var p in o)
{
  // won't see "p" as a property of o
  assert(p !== "p");
}
delete o.p; // does nothing, the property isn't configurable

o = {};
Object.defineProperty(o, "p", { get: function() { return 17; }, enumerable: true });
assert(o.p === 17);
for (var p in o)
{
  // assuming Object.prototype hasn't been modified
  assert(p === "p");
}

o = { p: 8675309 };
// p has descriptor { value: 8675309, writable: true, configurable: true, enumerable: true }
Object.defineProperty(o, "p", { writable: false });
// p has descriptor { value: 8675309, writable: false, configurable: true, enumerable: true }
o.p = 9; // p not writable, does nothing
assert(o.p === 8675309);
